# 谁是卧底 H5 多人游戏

一个手机优先的 H5 多人房间游戏。前端由 Express 静态托管，实时同步使用 Socket.IO，房间状态保存在服务端内存里。

## 本地运行

```bash
pnpm install
pnpm run dev
```

打开 `http://localhost:3000`。创建房间后，把 `/room/房间码` 链接发给其他玩家即可加入。

也可以运行 `outputs/start-local.command`。启动后：

- 本机打开 `http://localhost:3000`
- 同一 Wi-Fi 的手机或朋友打开 `http://你的电脑局域网IP:3000`

纯本地服务免费且不消耗 token，但只有这台电脑开着并运行服务时可访问；不在同一网络的人需要公网部署或隧道。

## 部署

稳定链接需要把服务部署到支持 Node.js 和 WebSocket 长连接的平台，例如 Render、Railway 或 Fly.io。临时 Cloudflare 链接只适合试玩，电脑关闭或进程停止后就不能继续玩。

部署命令：

```bash
pnpm install --frozen-lockfile
pnpm start
```

服务会读取平台提供的 `PORT` 环境变量。部署完成后，微信里分享 `https://你的域名/room/房间码` 即可。

如果使用 Render，可以直接使用仓库里的 `render.yaml` 创建 Web Service。

## 规则

- 最少 3 人开局。
- 3-4 人 1 名卧底，5-6 人 2 名卧底，7 人及以上 3 名卧底。
- 内置 255 组有迷惑性但不过度相近的词。
- 创建房间的第一位玩家默认是房主。
- 除房主外，其他玩家需要点击准备好了，房主才能开局。
- 发牌后只显示自己的词，不显示平民或卧底身份。
- 每局固定两轮发言，两轮结束后自动进入投票；平票或大房间第一次投错时，再加一轮发言。
- 每位玩家每轮 60 秒发言，最后 5 秒红色倒计时并播放提示音；可提前提交并自动轮到下一位。
- 发言支持文字和浏览器录音，发送后显示在公屏。
- 当前发言者可以跳过发言，系统会标注并发一条趣味弹幕。
- 开局后公屏是主要区域，玩家列表会压缩为当前发言者、自己和已淘汰玩家。
- 投票时先选择玩家，再点击确定提交；全员提交后系统自动公布答案。
- 投票阶段显示提交进度，房主可催促未提交玩家。
- 游戏内有房间聊天和弹幕区，淘汰玩家也可以观战发弹幕；炸弹弹幕有音效和动画。
- 游戏开始后会播放轻量背景音乐，浏览器或微信可能要求先点击页面后才允许播放声音。
- 同一个房间可继续下一把，胜者每人 +5 分，败者每人 -5 分并累计展示。
- 房主控制开局、跳过卡住的发言、结算和下一把。
- 5 人及以下：投票没投出卧底，卧底直接获胜。
- 6 人及以上：第一次没投出卧底会淘汰被投玩家，剩余玩家每人再发言一次；第二次仍没投出卧底，卧底获胜。
